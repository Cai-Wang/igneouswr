# WF1977 边界数据坐标系说明（2026-06-29 验证）

`boundaries/cls/winchester_floyd.json` 中存储的节点格式为 `[Nb/Y, Zr/TiO₂×0.0001]`，存储的是 **线性比值**（不是 log10 值）。

## 各字段含义

| 字段 | 含义 | 原始范围 | 图坐标 |
|------|------|---------|--------|
| nodes[n][0] | **Nb/Y** 比值 | 0.021 ~ 10.000 | 对数 Y 轴 (0.01 ~ 10) 或双对数 Y 轴 |
| nodes[n][1] | **Zr/TiO₂ × 0.0001**（ppm/ppm 体系） | 0.002 ~ 3.022 | 对数 X 轴 (0.001 ~ 10) 在双对数图中 |

## 两套坐标体系

### 体系 A：双对数（现有代码使用）
- X = Nb/Y, Y = Zr/TiO₂×0.0001
- 两轴都是对数
- 节点值直接使用，无需转换
- 范围：X 0.01~15, Y 0.001~10

### 体系 B：文献原始（Zr/TiO₂ wt% 比，线性 X）
- Y = Nb/Y（对数，0.01~10），直接用节点[n][0]
- X = Zr(ppm) / TiO₂(wt%)（线性，0~500） = 节点[n][1] × 10000
- **注意**：节点[n][1] = Zr/TiO₂×0.0001，乘以 10000 得到 Zr/TiO₂(wt%比)
- 实际计算：Zr/TiO₂wt% = Zr(ppm) ÷ TiO₂(wt%)，范围 20~30200
- 用 xlim(0,500) 截断，Phonolite 右上角（X≈30220）会超出图幅

## Zr/Ti 陷阱（用户提供的精确公式）

如果只有 Ti(ppm) 没有 TiO₂(wt%)：

```
Ti(wt%) = Ti(ppm) / 10000  （错误，Ti 不是 TiO₂）
TiO₂(wt%) = Ti(ppm) × 1.668 / 10000  （TiO₂/Fe 分子量比 79.866/47.867=1.668）
```

**用户给出的转换**（来自 LLM，未验证）：
```
X_轴 = Zr(ppm) / TiO₂(wt%) × 10⁻⁴
若只有 Ti(ppm)：X = Zr/Ti × 0.5995
```

验证：
- 玄武岩：Zr=100ppm, TiO₂=1wt% → 100/1 × 10⁻⁴ = 0.01
- 但在 0~500 线性 X 轴上，0.01 不合理
- 更合理的理解：X = Zr(ppm) / TiO₂(wt%)，玄武岩 X≈100，在 0~500 范围内

## 关键约束

现有的边界数据是从「双对数体系」手工数字化来的。如需在「线性 X 0~500、对数 Y 0.01~10」的文献坐标系中使用，必须：
1. 用节点[n][1] × 10000 作为 X 值
2. 用节点[n][0] 直接作为 Y 值（对数）
3. 设置 xlim(0,500), ylim(0.01,10), xscale('linear'), yscale('log')
4. **Phonolite 右上角会被截断**，因为该区域本就在 0~500 范围外

**或者更彻底**：直接从 GCDKit 的 .cvc/.csv 文件提取在「线性 X 坐标系」下数字化过的边界数据，替换现有 JSON。
