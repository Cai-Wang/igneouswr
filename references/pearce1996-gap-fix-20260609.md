# CLS-29 Pearce1996 底图缺口修复记录 (2026-06-09)

## 修复背景

GCDkit R 源码 `Pearce1996.r` 中，10 个填充多边形（`type="NULL"` 红色 fill）盖住了边界在 x=50 外的截断以及右垂线与顶斜线的不对齐。IgneousWR 采用纯线框模式后，这些缺陷暴露出来。

## 修复 1：下边界/中边界外推至 x=100

R 源码的分界线（lines11/lines12）只画到 x=50，但图幅 xlim=(0.01, 100)，x=50~100 之间无边界线。

在 log-log 空间用最后两点斜率外推：

| 位置 | 末点 (x,y) | 外推至 x=100 |
|------|-----------|-------------|
| 下边界 (lines11) | (50, 0.0781) | (100, 0.0944) |
| 中边界 (lines12) | (50, 0.2663) | (100, 0.3227) |

计算：
```python
# 下边界
log_x = [log10(2.81), log10(50)] = [0.4487, 1.6990]
log_y = [log10(0.0355), log10(0.0781)] = [-1.4498, -1.1073]
slope = (-1.1073 - (-1.4498)) / (1.6990 - 0.4487) = 0.2738
log10(y100) = -1.4498 + 0.2738 * (2.0 - 0.4487) = -1.0250
y100 = 10**-1.0250 = 0.0944

# 中边界
log_y = [log10(0.12), log10(0.2663)] = [-0.9208, -0.5747]
slope = (-0.5747 - (-0.9208)) / (1.6990 - 0.4487) = 0.2769
log10(y100) = -0.9208 + 0.2769 * (2.0 - 0.4487) = -0.4911
y100 = 10**-0.4911 = 0.3227
```

## 修复 2：右垂线顶部截断至顶斜线交点

R 源码：右垂线 lines14 顶部到 y=0.99，顶斜线 lines15 在 x=2.81 处 y≈0.957。在 R 中有填充多边形盖住这个 0.033 的超伸。

顶斜线在 log-log 空间的线段：(0.67, 0.2)→(5.52, 2)
在 x=2.81 处：
```
t = (log10(2.81) - log10(0.67)) / (log10(5.52) - log10(0.67))
  = (0.4487 - (-0.1739)) / (0.7419 - (-0.1739))
  = 0.6798
log10(y) = log10(0.2) + 0.6798 * (log10(2) - log10(0.2))
         = -0.6990 + 0.6798 * 1.0
         = -0.0191
y = 10**-0.0191 = 0.9569
```

## 修复 3：标签换行修复

R 源码中 `\n` 是真换行符，Python 之前用了 `\\n`（字面反斜杠-n），导致多行标签不换行。已改为真 `\n`。

foidite 的对齐从 `ha='center'` 改为 `ha='left'`（对应 R 的 `adj=c(0, 0.5)`）。
