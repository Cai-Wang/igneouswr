# 论文手稿数据复核工作流

> 当用户要求审校/补充英文 Discussion 草稿时，**先核对原始数据再回应**。
> 不能仅基于草稿中的数字做讨论——直接的错误会在草稿间传播致整段推论失效。

---

## 核心规则

**数据复核优先于内容生成。** 任何时候用户提交一篇包含具体数字的讨论/手稿草稿并要求填充或审校，第一件事是找到原始数据文件，逐一验算每个数字，**然后**再讨论怎样修改。

## 典型工作流

### 第一步：定位原始数据

- 查数据索引文件（`数据索引/*.md`）
- 定位到实际 Excel 文件路径
- 数据索引记录的文件名可能已不精确（实验室重命名等），先列目录确认 `ls`

### 第二步：提取并验算手稿中的关键数字

用 `pandas` 读入 Excel，针对手稿中出现的每个数值逐一验算：

**常见的手稿数字类型：**

| 数据类型 | 手稿中表述方式 | 验算方法 |
|---------|--------------|---------|
| 范围值 | "SiO₂ = 40.8–49.2 wt%" | `min()` / `max()` | 
| 均值 | "(avg. 43.9%)" | `mean()` |
| 比值 | "Eu/Eu* = 0.54–0.63" | 从原始元素计算或取已有计算值 |
| 标准化比值 | "(La/Yb)N = 8.7–14.5" | 用实际标准化值计算 |
| 比值比值 | "Nb/Th = 15–24" | 逐个样品计算 Nb/Th 再取范围 |

**注意要点：**
- 注意手稿中的数字是**单个样品**值还是**一系列样品的范围**（手稿常写 "0.54–0.63" 但实际是 range，也可能是作者的笔误）
- 区分手稿段落中出现的不同样品号的数字——24TJ02 的草稿可能是拼接了不同样品的数据
- 中文草稿和英文草稿的数字可能不同源，两者都要对比

### 第三步：分类处理差异

| 差异类型 | 示例 | 处理 |
|---------|------|------|
| **轻微偏离** | (La/Yb)N 草稿写 8.7–14.5，实际 9.1–15.2 | 直接更新草稿为实际值 |
| **严重错误** | Eu/Eu* 草稿写 0.54–0.63，实际 0.83–0.97（差40%） | 标记为关键错误，需要用户确认 |
| **数值无来源** | (Sm/Yb)N 草稿写 9–11，实际 4.4–5.4（差2倍） | 可能是混入了别的样品数据，标记并询问 |
| **手稿内部矛盾** | 英文草稿写 Eu/Eu*=0.54–0.63（强负异常），中文草稿写"无明显Eu异常" | 指出矛盾，确认哪个版本正确 |

### 第四步：系统性评估对推论的影响

严重错误的数值会连锁影响后续推论：

| 错误数值 | 原推论 | 实际值下的新推论 |
|---------|-------|----------------|
| Eu/Eu*=0.54–0.63 → 0.83–0.97 | "significant plagioclase fractionation" | 无明显斜长石分离结晶证据 |
| (Sm/Yb)N=9–11 → 4.4–5.4 | "garnet-bearing, >80 km" | 尖晶石-石榴石过渡带 (~60–80 km) |

### 第五步：给出结构化反馈

按以下格式报告验证结果：

```
## ✅ 能直接填的（数据准确）
| 参数 | 你写的 | 实际数据 |
|------|--------|----------|
| SiO₂ | ... | ... |

## ❌ 有重大冲突的（必须改）
**1. Eu/Eu\***
你写 X，实际 Y，差 Z%。

## ⚠ 需要确认的逻辑问题
...

## 需要用户填的 placeholder
- Mg#: 草稿留空 → 可填 XX-XX
```

### 第六步：报告界限

用户要求"能填多少填多少"时：
1. 列出所有**可精确填入**的值（以正确的实际数据替换草稿中的错误值）
2. 不编造没有的数据（同位素、缺的元素等）
3. 对 placeholder 给出可填的推荐值
4. 明确标注用户需要自己决定的部分（如推论框架是否需要因数值错误而调整）

## 常见陷阱（24TJ02 草稿复核经验）

- **Eu/Eu\** 极易写错**：人眼在蛛网图上容易把略微偏缺的 Eu 误判为"负异常"。实际数值可能接近 1。提交手稿前必须用计算值核对。
- **(Sm/Yb)N 易混入其他样品数据**：2 倍量级的差异往往来自引用了不同样品集的数据（如把更高压石榴石相样品的 (Sm/Yb)N 用于本样品）。
- **中英文草稿同源但数字可能不同**：用户可能同时维护中英文两个版本的 Discussion，两者之间数字要交叉对比。
- **Excel 中的计算比值优先信任**：21TJ02_merged.xlsx 中已有 ΣREE、Eu/Eu*、(La/Yb)N 等 40 个计算比值列，优先用它们——这些是自动计算的结果，比手写更可靠。
