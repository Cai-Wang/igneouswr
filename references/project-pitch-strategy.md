# IgneousWR 项目陈述策略

## 目标受众与叙事框架

### 对导师/课题组负责人
- **核心叙事**：科研基础设施工具。不再用"AI agent skill"措辞，改称"全岩地球化学批量成图引擎"
- **关键论据**：
  1. **效率量化**：2-3天 → 2分钟
  2. **标准化**：课题组统一图件风格（审稿人不问"为什么两张图字体不一样"）
  3. **可复现**：数据+脚本=完整计算记录
  4. **低门槛**：pip install，只需 Excel + Python

- **引用量预期**：
  - GCDkit (2004) 至今 1000+ 引用，说明了地学工具的长尾效应
  - AI 在地学渗透率当前 ~5%，3-5 年到 30-50%
  - IgneousWR 是 API-native 设计，天然可被 AI 调用——这是大模型时代的代差
  - 当前零竞品窗口期 1-2 年
  - 引用预测：50 → 500 → 1000+

- **关键论据**：不用学 R、不用装 GCDkit、不用一张一张点

### 对潜在投资人/基金申请
- **核心叙事**：地学领域 AI 渗透的刚需基建
- **关键论据**：窗口期 + 引用前景 + 可投稿期刊（Computers & Geosciences, G³ 等）

## 对标 GCDkit 的标准框架

| 维度 | GCDkit (R) | IgneousWR (Python) |
|------|-----------|-------------------|
| 安装 | 需 R + 大量依赖 | pip install |
| 操作 | GUI 手动点选 | 一行代码一键出图 |
| 输出 | GUI 截图，风格各异 | 统一风格 PNG + HTML |
| 学习成本 | 中等 | 低 |
| AI 兼容 | ❌ GUI 无法被 AI 调用 | ✅ API 设计，天然可被 Claude/ChatGPT 等调用 |

## PPT 模板建议

- 深色主题（深色背景 + 青蓝强调色）
- 13 页为宜：封面 → 项目定义 → 已有成果 → 对标 → 痛点 → 解决方案 → 人力需求 → 人力方案 → 承诺 → Roadmap → 学术发表 → 引用前景 → 结尾
- 关键数字用大号加粗突出（2-3天→2分钟、19张、1000+引用）
- 引用曲线分三期图示（一期50→二期500→三期1000+）

---

## 常见陷阱

- ❌ 用"AI Agent skill"措辞对导师/传统地学者——导师听到"AI"只会觉得不稳定
- ✅ 用"批量成图引擎"、"科研基础设施工具"定位
- ❌ 只讲技术不讲引用前景——导师在意的是人力投入是否产出论文
- ✅ 强调"AI 时代窗口期"——越早发论文引用累积周期越长
