地化出图工具面临的根本矛盾：**agent 对接是效率最优**（一句话出图），但**纯无头缺少视觉冲击力**（用户看不到力量）。

- ✅ 第二阶段（HTML 报告）：**已实施**
- ❌ 第三阶段（API + 付费层）：规划中

**igneous-geochemistry skill** 走 agent 优先路线（自然语言驱动），但补上 visual showcase 层。

## 三阶段产品路线

### 第一阶段（已完成）：Agent Skill

```python
# 用户说一句就出图
"读这个 Excel，出 TAS 和 REE 图，风格按 Lithos"
```

核心能力已具备：数据读取、岩性判定、推荐入口、风格预设。

问题：看不到——用户收到的是散文件 PNG。

### 第二阶段（已实施 2026-05-08）：HTML 图集报告

`plot_recommended()` 出图完毕后自动在输出目录生成 `report_YYYYMMDD.html`：

- 所有图以 2 列网格排列，按 **4 个方向分组**（分类 → 源区 → 演化 → 构造），单击缩略图查看原图（lightbox 放大）
- 顶部显示数据来源、样品数量、岩性判定
- 总览卡片：推荐总数 / 成功数 / 跳过数
- 跳过的图透明列出（缺哪些元素）
- 可直接发给合作者，无需装 Python
- 自包含 HTML，不需要服务器、不需要数据库、不需要前端框架

实现方式：`_style.py` 中的 `generate_report_html(success, skipped, gd=gd, out_dir=final_dir, rock_type=rock_type)`，由 `plot_recommended()` 末尾自动触发。

也支持手动调用：
```python
from whole_rock_core import generate_report_html
result = plot_recommended(gd)
generate_report_html(result['success'], result['skipped'], gd=gd)
```

### 第三阶段（愿景）：API + 付费层

```
┌─ 免费层 ─────────────────────────────┐
│ agent skill（开源，自用）             │
├─ 付费增值 ────────────────────────────┤
│ ① 风格预设库（Nature/Lithos/GCA）    │
│ ② HTML 报告自动生成                  │
│ ③ 预设图件底图在线预览              │
│ ④ OpenAPI 接口（按次计费）          │
└──────────────────────────────────────┘
```

API 层的核心卖点不是"画图软件"，而是**标准化出图能力**：
- 地学专业壁垒（图件分类体系、场界数据、命名规范）
- 风格一致性（自动符合期刊要求）
- 可被其他 AI agent 调用

## 多用户个性化方案

风格管理体系分三层，不搞 GUI：

### 层 1：风格预设（已实现）

用户/agent 说一句就切换：
```python
gc.set_style_preset('nature')
```

### 层 2：个性化调色板+分组（已实现）

```python
groups = ['玄武岩','玄武岩','安山岩','安山岩','流纹岩']
gc.scatter_groups(ax, x, y, labels, groups=groups)
```

### 层 3：用户自定义预设文件（未来）

每个用户可存一个 `.igneous.style` YAML 文件，定义自己的默认值：
```yaml
preset: nature
overrides:
  MK_SIZE_SINGLE: 80
  LEGEND_LOC: 'lower right'
```

agent 自动检测并应用。

## 为什么不做传统 GUI

- GUI 需要"上传文件→下拉选择→点按钮"，用户想要"一句话搞定"
- 地学用户不是不会用 GUI，而是数据探索需要反复调参，GUI 的操作成本远高于自然语言
- 但纯无头在展示/分享场景中体验差——这就是 HTML 报告要补的

## 关键设计决策

| 决策 | 选择 | 原因 |
|------|------|------|
| 交互入口 | agent 自然语言 | 效率最优 |
| 交付形式 | PNG + HTML 报告 | 兼顾视觉 |
| 风格配置 | 代码内预设，不进数据库 | 零运维 |
| 收费点 | API + 风格预设库 | 面向开发者/AI agent |
| 颜色策略 | 分组着色 + 合并图例 | 解决真实痛点（样品分组） |

## 反模式记录（2026-05-25）

以下方向已被评估并排除，不应重复提出：

### ❌ 不新增 excel_tools/ 独立脚本目录

意图：把标准化计算、比值提取、判别图坐标预计算等固化到可独立调用的 CLI 脚本中，供用户在终端直接运行。

**排除原因**：这个 skill 的设计目标就是"让 AI 做事，人不动手"。独立脚本让人记住路径传参调用，等于退回到用户亲自按计算器的模式。所有数值计算已经由 Python 代码（`_chem.py`、`_normalize.py`、`merge_excel.py` 的 `compute_ratios()`）在管线内部自动完成，无需额外出口。

**正确做法**：如果用户需要纯数值输出（不画图），AI 应该在内部调用已有 Python 模块计算结果并呈现给用户，而不是让人去跑脚本。

### 当前 skill 的波动边界

数值计算层（标准化、FeOt、比值、判据坐标、三元图变换）已全部固化在 Python 代码中，模型差异不会影响计算结果。模型波动仅影响 AI 推理范畴：

- 图件选择（出哪些图）
- 风格参数（点大小、颜色、图例位置）
- 结果判读（点在某个区的构造含义）

推理范畴的问题待后续单独讨论，不与数值计算层混为一谈。
